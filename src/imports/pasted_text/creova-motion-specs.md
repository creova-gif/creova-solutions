## PART 3: TRANSITIONS – SEAMLESS & CINEMATIC

### Scroll Transitions

| Trigger | Effect |
|---------|--------|
| **Section to section** | **Morph dissolve** – current section fades into particles, next section assembles smoothly |
| **Between cards** | **Slide + subtle blur** – elegant, like cinematic rack focus |
| **Modal open** | **Expand** – button expands into modal with smooth scaling, no shatter |

### Page / View Transitions

| Trigger | Effect |
|---------|--------|
| **Link click** | **Fade + scale** – page fades out slightly, scales to 0.98, new page scales in |
| **Back button** | **Reverse fade** – same as forward but direction-aware |
| **External link** | **Portal effect** – screen shows "opening doorway" animation before leaving |

### Loading Transitions

| Element | Effect |
|---------|--------|
| **Initial page load** | **Logo assembly** – parts of logo glide in from edges, snap together with soft glow |
| **Lazy-loaded section** | **Fog reveal** – section starts blurred (blur 8px), clears on scroll into view |
| **Image loading** | **Progressive blur** – starts blurred/soft, sharpens smoothly on load |

### Video Specifications

| Aspect | Requirement |
|--------|-------------|

Aspect ratio 16:9. Cinematic lighting. Shallow depth of field where appropriate."*

## PART 7: PERFORMANCE & ACCESSIBILITY

| Aspect | Requirement |
|--------|-------------|
| **WebGL fallback** | If Three.js fails, degrade to CSS animations + static SVGs |
| **Reduced motion** | Respect `prefers-reduced-motion` – disable all non-essential motion |
| **Mobile optimization** | Reduce particle count, lower 3D quality, replace complex diagrams with simplified versions |
| **Lazy loading** | Load 3D scenes and videos only when in viewport (Intersection Observer) |
| **Target FPS** | 60fps on desktop, 30fps minimum on mobile |
